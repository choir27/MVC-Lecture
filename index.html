Richard's MVC Lecture

What is a MVC?

  • It is a way to separate different parts of your backend code
  • It does this by categorizing them into four different categories:
  • Routers, Controllers, Models, Views

    • Routers

    • Controllers

    • Models

    • Views

    Routers

  • Hears the /GET request that the client makes
  • Looks for the controller respective to the /GET request
  • Responds back to the client with that controller

    • Controllers

  • Is told by the Routers what to do
  • Tells the Views what to show to the client
  • Tells the Models what to do with the data
  • Responds back to the Routers with the updated Views and Models

    • Models

  • Is told by the Controllers what to do with the data
  • Decides what to send to the data
  • Responds back to the Controllers with the updated data

    • Views

  • Is told by the Controllers what to show to the Client
  • Shows the website content, styles, and actions
  • Responds back to the Controllers with the updated HTML/CSS/JS

    • Pros of using MVC model:

  • Keeps the code clean
  • Keeps the code and files organized
  • Makes it significantly easier to work in a team

    • What is Mongoose?

  • It helps connect our server to our database, like MongodB
  • Maintains data relationship between the server and database
  • Works with schemas, which are just a map of keys and types, like a blueprint
